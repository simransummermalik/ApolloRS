# Architecture Decision Records

ADRs capture decisions that affect semantic fidelity, provenance,
reproducibility, crate boundaries, or research claims.

Each ADR contains status, context, decision, alternatives, consequences, risks,
and validation.  Accepted ADRs are not rewritten to hide changed thinking; a
new ADR supersedes them and links the earlier record.

Statuses are `Proposed`, `Accepted`, `Superseded`, or `Rejected`.
